## 🔴 먼저 알아야 할 blur 특유의 심사 리스크 3가지

App Store 심사(App Review Guidelines)에서 blur 같은 **UGC 사진 SNS + 소셜 로그인** 앱이 특히 걸리는 지점.

### 1. Guideline 4.8 — Sign in with Apple 필요
카카오·구글 같은 **제3자 소셜 로그인을 제공하면**, 그에 준하는 프라이버시 보호 로그인(= **Sign in with Apple**)을 함께 제공해야 함.
- 현재 blur는 카카오/구글 **전용** → **Sign in with Apple 추가가 사실상 필수**.
- Supabase는 Apple provider 지원 → Expo 재구현 때 `signInWithApple` 추가 + Apple Developer에서 Service ID/Key 설정.
- ⚠️ 이건 자료가 아니라 **개발 작업** → 로드맵 Phase 1에 포함.

### 2. Guideline 1.2 — UGC 앱 안전장치 (5종 전부 필요)
사용자가 사진·글을 올리는 앱은 다음을 **모두** 갖춰야 함:
- [x] 신고 기능 (reports 테이블·4경로 구현됨)
- [x] 차단 기능 (block 구현됨)
- [x] 게시된 EULA/약관 (terms.html)
- [x] 이용약관 동의 절차 (가입 문구)
- [ ] **부적절 콘텐츠 필터링 + 24시간 내 신고 처리 약속** — 자동/수동 모더레이션 계획을 심사노트에 명시해야 함 (아래 "모더레이션 계획" 항목)

### 3. Guideline 2.1 — 로그인 전용 앱의 심사용 데모 계정
로그인해야 앱을 볼 수 있는데 blur는 **OAuth 전용**이라 Apple 심사자가 들어올 방법이 없음.
- 대응 A(간단): App Review 정보에 **작동하는 구글 테스트 계정** 이메일/비번 제공 (전용 지메일 1개 생성 권장).
- 대응 B: 심사용 데모 로그인 경로(고정 계정) 임시 추가.
- → Phase 2에서 결정. **전용 구글 계정 생성이 가장 무난.**

## Phase 0 — 지금 바로 가능 (바이너리 불필요)

### 0-1. Apple Developer Program 등록
- [ ] Apple Developer Program 가입 — **개인(Individual) 자격, 연 $99 (약 14만원)**
- [ ] 결제 후 활성화까지 24~48시간 (신원 확인 요청 올 수 있음)
- [ ] 실기기가 아이폰이라 본인 검증 유리
- 참고: 개인 자격이면 D-U-N-S 번호 불필요. 개발자 이름이 판매자로 공개됨(법인명 원하면 조직 등록 필요 — 개인으로 시작 권장).

### 0-2. 앱 기본 정보 확정
- [ ] **Bundle ID** 확정 — 예: `com.kim.blur` 또는 `star.kdh96980919.blur` (한번 정하면 변경 어려움)
- [ ] **앱 이름(App Name)** — 최대 30자. "blur" 단독은 상표/중복 검색 필요. 예: "blur — 하루 한 번의 사진"
- [ ] **부제(Subtitle)** — 최대 30자. 예: "흐릿하게 만나는 사진 일기"
- [ ] **카테고리** — Primary: Social Networking / Secondary: Photo & Video
- [ ] **연령 등급** — UGC 사진 공유라 **17+** 가능성 높음(모더레이션 수준에 따라). App Store Connect 설문으로 결정

### 0-3. 텍스트 메타데이터 (한국어 + 영어 권장)
- [ ] **설명(Description)** — 최대 4000자. 컨셉·주요 기능·하루 1회 흐림 해제 스토리
- [ ] **키워드(Keywords)** — 최대 100자, 쉼표 구분. 예: 사진,일기,친구,SNS,블러,하루,감성,다이어리
- [ ] **프로모션 텍스트(Promotional Text)** — 최대 170자, 심사 없이 수정 가능
- [ ] **지원 URL(Support URL)** — 필수. 문의 페이지 or 랜딩(invite/) 재사용 가능
- [ ] **마케팅 URL** — 선택. 라이브 랜딩 재사용
- [ ] **개인정보처리방침 URL** — 필수. `legal/privacy.html` 라이브 주소 사용 ✅ 이미 있음
- [ ] **저작권 표기** — 예: "2026 Kim Donghyun"

### 0-4. 그래픽 에셋
- [x] **앱 아이콘 1024×1024 PNG** ✅ (2026-07-20) — `assets/appstore/icon-1024.png`. 김이 만든 스카이블루+민트 블러 이미지 사용. RGB(알파 없음)·꽉 찬 정사각·둥근 모서리 없음 확인. **앱 전반 기호도 이걸로 통일**: 파비콘(favicon-32·icon-192)·PWA 매니페스트(192/512)·iOS 홈화면(apple-touch-icon-180)·푸시 알림 아이콘·SW 캐시(v19) 모두 교체. 옛 `icon.svg`(어두운 원) 참조 전부 제거. ⏳ 라이브 반영은 배포 필요
- [ ] **스크린샷 (iPhone 6.9"/6.7")** — 1320×2868 또는 1290×2796, **2~10장**. 온보딩·오늘 탭·업로드·blur 해제·룸·DM 등 대표 화면. 캡션 오버레이 넣으면 전환율↑
- [ ] (앱이 iPad 지원이면) iPad 스크린샷 — 아이폰 전용으로 낼 거면 불필요
- [ ] (선택) 앱 미리보기 동영상 15~30초

### 0-5. 정책·모더레이션 문서
- [x] 개인정보처리방침 (privacy.html) — ⚠️ App Privacy "영양성분표"용으로 **수집 항목 재점검 필요**(아래 0-6)
- [x] 이용약관 (terms.html)
- [ ] **모더레이션 계획** 문서화 — 신고 접수→검토→24h 내 조치 흐름, 금지 콘텐츠 정의, 반복 위반 밴(banned 시스템 이미 있음). 심사노트에 요약 첨부
- [ ] 문의 연락처 명시 — 202501630@inu.ac.kr (약관에 이미 있음)

### 0-6. App Privacy (데이터 수집 신고) 설문 준비
App Store Connect에서 수집 데이터를 신고해야 함. blur가 수집하는 것 미리 정리:
- [ ] **사진/영상** (사용자 콘텐츠) — 앱 기능용
- [ ] **연락처** — ⚠️ 연락처 친구찾기 기능. 원본 미저장·해시만 저장하지만 **"연락처 접근"은 신고 대상**. "친구 추천" 목적으로 명시
- [ ] **이메일/이름** (계정) — OAuth 프로필
- [ ] **사용자 ID** (handle, uuid)
- [ ] **푸시 토큰** (기기 식별)
- [ ] 각 항목: 추적(Tracking) 여부 = **No**(광고·제3자 추적 없음), 사용자와 연결됨(Linked) 여부 표시
- → 이 답변이 privacy.html 내용과 **일치해야** 함. 불일치 시 리젝 사유.
